Why Lumine

A familiar foundation with the important parts left open.

Hackable by design

Shape the editor with packages, keymaps, commands, themes, and direct web-platform tools.

Lineage

Lumine continues the idea of an editor you can take apart and rebuild around your workflow, with the project stewarded by the community that uses it.

Build

Build from source

Lumine follows the Pulsar-style source workflow: clone the repo, use the Node version from .nvmrc, let Corepack provide Yarn, then build and start the app.

Recommended local flow

Runs the app from source 
# Clone the source
git clone https://github.com/lumine-code/lumine.git
cd lumine

# Use the project Node version from .nvmrc
nvm install

# Enable Yarn through Corepack once for this Node version
corepack enable

# Install, rebuild native modules, and launch
yarn install
yarn build
yarn start

# Optional: produce platform binaries in binaries/
yarn dist

Corepack

corepack enable is the preferred way to make Yarn available. Run it once per Node version if yarn is not already present.

Dev resource path

If Lumine is already installed, point the installed app at this source checkout and launch it in dev mode.

# PowerShell
$env:ATOM_DEV_RESOURCE_PATH = (Get-Location).Path
lumine --dev .

# macOS/Linux
ATOM_DEV_RESOURCE_PATH="$PWD" lumine --dev .
